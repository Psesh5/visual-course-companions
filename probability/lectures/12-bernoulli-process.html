Lecture 12 · The Bernoulli process

Flip a biased coin once per time slot, forever. Each slot is an independent success with probability p and a failure with probability 1−p. That whole stream of i.i.d. coin flips — nothing more — is the Bernoulli process. From it fall three of the most useful distributions in all of probability. ▶ watch on OCW (Tsitsiklis 6.041)

What the process is

Time is chopped into slots. In each slot we run one independent Bernoulli trial: Xᵢ = 1 (success / arrival) with probability p, and Xᵢ = 0 with probability 1−p. The trials share the same p and never look at one another — independent and identically distributed. That is the entire definition. There is no hidden state, no momentum, no clock that "remembers." Everything interesting below is just a different question asked of this one stream of coin flips.

Fact 1 — counting arrivals: the Binomial

Ask: how many successes in the first n slots? Call it S = X₁ + ⋯ + Xₙ. Each way to get exactly k successes has probability pᵏ(1−p)ⁿ⁻ᵏ, and there are C(n,k) arrangements of which slots succeeded, so 

P(S = k) = C(n, k) · pᵏ · (1−p)ⁿ⁻ᵏ          k = 0, 1, …, n      (Binomial)

E[S] = n·p           Var(S) = n·p·(1−p)

The mean E[S] = np is the cleanest fact in the subject: linearity of expectation adds up n coins each worth p on average, and it never even needs independence. Switch the bottom plot to counts, run thousands of length-n histories, and the histogram of S settles right onto that black Binomial curve, centered at np.

Fact 2 — waiting between arrivals: the Geometric

Now ask: how many slots until the next success? Call the inter-arrival gap T. To wait exactly k slots you need k−1 failures in a row and then a success: 

P(T = k) = (1−p)ᵏ⁻¹ · p              k = 1, 2, 3, …            (Geometric)

E[T] = 1 / p          Var(T) = (1−p) / p²

The mean gap 1/p is the natural reciprocal of the rate: if a success lands one slot in ten (p = 0.1) you wait about ten slots on average. Switch the bottom plot to gaps: we run one very long path, measure every spacing between consecutive arrivals, and histogram them — a clean decaying staircase sitting exactly on the Geometric PMF. Notice the gap is always ≥ 1; the very first arrival time is itself Geometric(p) too.

Fact 3 — memorylessness: the coin has no past

Suppose you have already waited s slots with no success. How much longer must you wait? The surprising answer: exactly as long as if you had just started. Formally, 

P(T > s + t | T > s) = P(T > t)          for all s, t ≥ 0

because  P(T > m) = (1−p)ᵐ ,  so  (1−p)^(s+t) / (1−p)^s = (1−p)^t.

The past s failures cancel out completely. The coin has no memory — waiting longer does not make a success "more due." The Geometric is the only discrete distribution with this property, and it is a direct consequence of the trials being independent: the slots you already spent tell you nothing about the slots to come. This is the same reason gamblers' "it's been red eight times, black is overdue" reasoning is wrong. Each spin, like each slot, starts fresh.

The bridge — squeeze the slots and you get the Poisson process

The Bernoulli process lives in discrete time. But real arrivals — photons, phone calls, radioactive decays — happen in continuous time. Take the limit: chop each second into more and more tiny slots, let the slot width shrink, and let p → 0 while holding the expected rate n·p → λ fixed. Two things converge:

  • the count in a fixed window, Binomial(n, p) with np = λ, converges to Poisson(λ) — the famous rare-events limit; and
  • the gap, Geometric(p), rescaled by the slot width, converges to the Exponential distribution — which is memoryless in continuous time, inheriting Fact 3 directly.

That continuous-time limit is the Poisson process (Lecture 13). Everything you are seeing here has an exact analog there: Binomial → Poisson, Geometric → Exponential, discrete memorylessness → continuous memorylessness. Drag p small and n large in the counts view and you can watch the Binomial bars already starting to look Poisson — the bridge is visible before you ever cross it.
