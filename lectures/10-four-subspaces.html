Lecture 10 · The four fundamental subspaces

Every m×n matrix A creates four subspaces. Two live in the domain Rn (row space & nullspace), two live in the codomain Rm (column space & left nullspace). In each space the pair is orthogonal and their dimensions add to the whole. ▶ watch on OCW
The matrix. We use A = [[1,2,3],[1,1,1],[2,3,4]], a 3×3 matrix of rank 2. Because m = n = 3, all four subspaces sit inside the same R3, so you can actually see the right angles. Row 3 = Row 1 + Row 2, and the columns satisfy c1 − 2c2 + c3 = 0, so both rows and columns are dependent.

The big picture

Left figure — the domain Rn (here R3). The blue plane is the row space C(AT): all combinations of the rows of A. The green line is the nullspace N(A): every vector x with Ax = 0. The line is perpendicular to the plane — that is the whole point. A vector solves Ax = 0 exactly when it is orthogonal to every row, i.e. orthogonal to the row space. Dimensions: r = 2 for the row space, n − r = 1 for the nullspace, and 2 + 1 = 3 = n.

Right figure — the codomain Rm (here R3). The orange plane is the column space C(A): all vectors b you can reach as Ax. The red line is the left nullspace N(AT): every y with ATy = 0, equivalently yTA = 0. Again the line is perpendicular to the plane: y kills A from the left exactly when it is orthogonal to every column. Dimensions: r = 2 column space, m − r = 1 left nullspace, summing to m = 3.

Why orthogonality is forced. If Ax = 0 then each entry of Ax is a row of A dotted with x, and all those dots are zero — so x ⟂ every row. The nullspace is the orthogonal complement of the row space inside Rn. The same argument on AT gives column space ⟂ left nullspace inside Rm. Rotate the figures: each green/red line pierces its plane at a right angle through the origin.

Worked math — finding all four subspaces by elimination

Start from A = [[1,2,3],[1,1,1],[2,3,4]] and run Gauss–Jordan. Subtract row 1 from row 2, and twice row 1 from row 3: 

A = | 1  2  3 |      R2−R1, R3−2R1     | 1  2  3 |      R3+R2          | 1  2  3 |
    | 1  1  1 |   ───────────────►     | 0 −1 −2 |   ──────────►       | 0 −1 −2 |
    | 2  3  4 |                        | 0 −1 −2 |                     | 0  0  0 |

reduced row echelon form  R =  | 1  0 −1 |     pivots in columns 1 and 2  ⇒  rank r = 2
                               | 0  1  2 |     free column: 3
                               | 0  0  0 |

1 · Row space C(AT) ⊂ R3, dim = r = 2. The nonzero rows span it. A clean basis is the two pivot rows of R — or, equivalently, the two independent original rows: {(1, 2, 3), (1, 1, 1)}. (Row 3 = Row 1 + Row 2 contributes nothing.)

2 · Nullspace N(A) ⊂ R3, dim = n − r = 1. Solve Rx = 0. With free variable x₃ = 1: from row 2, x₂ = −2x₃ = −2; from row 1, x₁ = x₃ = 1. Special solution {(1, −2, 1)} — exactly the column relation c1 − 2c2 + c3 = 0. Check A·(1,−2,1) = 0. ✓

3 · Column space C(A) ⊂ R3, dim = r = 2. Pivots landed in columns 1 and 2, so the original columns 1 and 2 form a basis: {(1, 1, 2), (2, 1, 3)}. Column 3 = 2·col 2 − col 1 lies in their span. (Use original, not reduced, columns — elimination changes the column space.)

4 · Left nullspace N(AT) ⊂ R3, dim = m − r = 1. We need yTA = 0. The same elimination that zeroed row 3 records the recipe: row 3 − row 2 − row 1 = 0, i.e. −R1 − R2 + R3 = 0, giving {(1, 1, −1)}. Check (1,1,−1)·A = 0 i.e. AT·(1,1,−1) = 0. ✓

Counting (the fundamental theorem). domain R³: r + (n−r) = 2 + 1 = 3; codomain R³: r + (m−r) = 2 + 1 = 3. Row space ⟂ nullspace and column space ⟂ left nullspace, so each space splits cleanly into two perpendicular pieces.

Strang's actual lecture example — when m ≠ n

The 3×3 matrix above lets all four subspaces share one R³ so you can see the right angles. But in the lecture Strang works a rectangular matrix, and that is where the four spaces genuinely live in two different ambient spaces. His matrix is 3×4 (m = 3, n = 4), rank 2, with two identical rows: 

A = | 1  2  3  1 |        Gauss–Jordan on  [ A | I ]        R = | 1  0  1  1 |    rank r = 2
    | 1  1  2  1 |   ─────────────────────────────────►        | 0  1  1  0 |    pivots in cols 1, 2
    | 1  2  3  1 |        (rows 1 and 3 are equal)              | 0  0  0  0 |    free cols 3, 4

Now the two pairs sit in different R's, and the dimensions are no longer symmetric:

  • Row space C(Aᵀ) ⊂ R⁴, dim r = 2. Basis = the nonzero rows of R: {(1,0,1,1), (0,1,1,0)}.
  • Nullspace N(A) ⊂ R⁴, dim n − r = 2. One special solution per free column: {(−1,−1,1,0), (−1,0,0,1)}. (Check 2 + 2 = 4 = n.)
  • Column space C(A) ⊂ R³, dim r = 2. Basis = original pivot columns 1, 2: {(1,1,1), (2,1,2)}. (col 3 = col 1 + col 2, col 4 = col 1.)
  • Left nullspace N(Aᵀ) ⊂ R³, dim m − r = 1. Basis = {(−1,0,1)}: it says row 3 − row 1 = 0. (Check 2 + 1 = 3 = m.)

Slide the rank below to watch the four dimensions (r, n−r, r, m−r) rebalance while always summing to n = 4 on the left and m = 3 on the right — the fundamental theorem, live.

How the left nullspace falls out of elimination — the matrix E

The row space, nullspace and column space all "jump out" of R. The left nullspace does not — you have to keep a record of the elimination steps. Strang's trick: run Gauss–Jordan on the augmented [ A | I ]. Whatever row operations turn A into R also turn I into a matrix E with E·A = R. For this example: 

E = | -1   2   0 |      E · A = R      Every ZERO row of R came from a row of E
    |  1  -1   0 |   ───────────────►  killing a combination of A's rows.
    | -1   0   1 |                     R's row 3 is zero  ⇒  E's row 3 = (-1, 0, 1)  ∈  N(Aᵀ).

The rows of E that produce zero rows of R are exactly a basis for the left nullspace. Here only the last row of R is zero, so the left nullspace is one-dimensional, spanned by (−1, 0, 1) — and indeed (−1,0,1)·A = 0, i.e. row 3 − row 1 = 0. (In the square invertible case R = I has no zero rows, so E = A⁻¹ and the left nullspace is just {0}.)

Elimination keeps the row space but changes the column space

A point Strang stresses: row operations are combinations of the rows, so they leave the row space unchanged — that is why a basis is read straight off the rows of R. But they scramble the columns, so the column space changes. Concretely (1,1,1) is column 1 of A, so it is in C(A) — yet R has a zero in its bottom row, so (1,1,1) is not in C(R).
