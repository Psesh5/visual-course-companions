Lecture 5 · Transposes, permutations, and the spaces Rn

Permutation matrices just reorder coordinates; transposes flip rows and columns; and a vector space (or subspace) is a set closed under adding vectors and scaling them. ▶ watch on OCW
Part 1 — Permutation matrices. A permutation matrix P is the identity with its rows shuffled. Multiplying P·v just reorders the entries of v. Pick one of the six 3×3 permutations below; the blue arrow is the original v=(1,2,3) and the orange arrow is P·v. Notice the orange arrow is always the same length — permutations only rename axes.
Choose P:
Key fact: every permutation matrix is orthogonal, so PT = P-1. Undoing a shuffle is the same as transposing it. There are n! = 6 permutation matrices for n = 3.
Where permutations come from: PA = LU. Strang's framing: elimination gives A = LU only when no zero ever lands in a pivot. If a zero shows up in the pivot position, you must exchange rows — and that row exchange is exactly a permutation matrix P. Account for every needed exchange up front and elimination always succeeds: PA = LU. For almost every matrix you need none, so P = I; for the handful that do, P picks the good row order. (MATLAB exchanges rows even when algebra doesn't require it, to dodge tiny pivots that are numerically bad.)
How many permutation matrices? n! — that is n·(n-1)·(n-2)···(3)(2)(1), the number of ways to reorder n rows. The identity (do-nothing) counts as one of them.

Part 3 — Transpose & symmetry. The transpose AT reflects a matrix across its main diagonal: (AT)ij = Aji — the row and column numbers just swap. A rectangular matrix flips shape: Strang's R is 3×2, so RT is 2×3 ("short and wider"). A matrix is symmetric when it equals its own transpose, A = AT.
How to build a symmetric matrix: RTR. A rectangular R is "absolutely far from symmetric" — it isn't even square. But RTR is always symmetric. Strang does it with the actual numbers below. Hover any off-diagonal cell of the product to see its mirror cell light up — they are forced equal.
Why it must be symmetric (no numbers needed). Transpose the product using the order-reversing rule (AB)T = BTAT: (RTR)T = RT(RT)T = RTR, since (RT)T = R (flipping across the diagonal twice gives R back). The product equals its own transpose ⇒ symmetric.
Part 4 — Column space: how a matrix makes a subspace. Strang's "central idea, the great example." Take the columns of a matrix and form all their linear combinations x₁·col₁ + x₂·col₂ — that set is automatically a subspace, the column space C(A). His matrix A has columns (1,2,4) and (3,3,1) in R³. Those two vectors aren't on one line, so their combinations fill a whole plane through the origin — not all of R³, more than a line. Drag the two sliders to land on any point of that plane.
What you'd get otherwise. If the two columns had landed on the same line, the column space would shrink to just that line. Pile up columns that point every which way and the column space grows to all of R³. In R10 you can't picture it, but the rule is identical: combinations of the columns form "some flat thing through the origin" — a subspace. The intuition

Permutations are just relabeling. When you multiply P·v you are not stretching or rotating space in any meaningful geometric way — you are swapping which coordinate is called x, which is y, which is z. That is why |P·v| = |v| always, and why doing it twice in reverse (transpose) puts everything back: PTP = I.

A subspace is "closed" world. Think of a flat plane through the origin as a tabletop. Slide along it (add two vectors on it) and you are still on the table. Push out from the origin or pull back through it (scale, including by 0 and by negatives) and you are still on the table. The instant a set forces you off — say an offset plane that does not pass through 0 — it is not a subspace, because scaling any member by 0 must land on the zero vector, and an offset plane does not contain 0.

Why 0 must be in every subspace. Closure under scaling says c·v stays in the set for every scalar c. Take c = 0: then 0·v = 0 must be in the set. So the zero vector is unavoidable — it is the smallest possible subspace, {0} itself.

Worked math

(a) The two closure axioms, checked on a plane. Let S be the plane x + y + z = 0 through the origin. Take two members:

  u = (1, -1, 0)   check: 1 + (-1) + 0 = 0   ✓ in S
  w = (2,  1, -3)  check: 2 +   1  + (-3) = 0 ✓ in S

  Closure under addition:   u + w = (3, 0, -3),   sum = 3 + 0 + (-3) = 0   ✓ still in S
  Closure under scaling:    5·u   = (5, -5, 0),   sum = 5 + (-5) + 0 = 0   ✓ still in S
  Contains zero:            0      = (0, 0, 0),    sum = 0                  ✓ in S

  All three hold for every choice ⇒ S is a subspace (a 2-D plane through the origin).

(b) Why an offset plane fails. Let T be the parallel plane x + y + z = 1 (does not pass through 0).

  p = (1, 0, 0)  check: 1 + 0 + 0 = 1  ✓ in T
  q = (0, 1, 0)  check: 0 + 1 + 0 = 1  ✓ in T

  Closure under addition:  p + q = (1, 1, 0),  sum = 1 + 1 + 0 = 2 ≠ 1   ✗ NOT in T
  Closure under scaling:   0·p   = (0, 0, 0),  sum = 0 ≠ 1              ✗ NOT in T (no zero vector)

  Either failure alone is fatal ⇒ T is NOT a subspace. The same logic rules out any line or
  plane that misses the origin.

(b′) Strang's quarter-plane: closed under + but not scaling. The first quadrant of R² (all vectors with both components ≥ 0) is a tempting non-example.

  Take v = (3, 2)  and  w = (5, 6),  both in the quarter-plane.

  Closure under addition:   v + w = (8, 8)        ✓ still in the quarter-plane (adding two ≥0 stays ≥0)
  Closure under scaling:    (-5)·v = (-15, -10)    ✗ NOT in it — a negative scalar throws you out

  Addition alone is not enough. A subspace must be closed under EVERY scalar, including negatives
  and 0 ⇒ the quarter-plane is NOT a subspace.

(b″) The column space is a plane through 0. Strang's matrix A has columns c₁=(1,2,4), c₂=(3,3,1). The column space is every combination x₁c₁ + x₂c₂.

  x=(0,0):  0·c₁ + 0·c₂ = (0,0,0)      ✓ contains the origin
  x=(1,0):  c₁          = (1, 2, 4)     a point on the plane
  x=(0,1):  c₂          = (3, 3, 1)     another point
  x=(1,1):  c₁ + c₂     = (4, 5, 5)     the sum is on the plane too
  x=(2,-1): 2c₁ − c₂    = (-1, 1, 7)    any combination stays on the plane

  c₁ and c₂ are not multiples of each other ⇒ their combinations sweep out a 2-D plane through 0,
  the column space C(A). It is more than a line, less than all of R³.
