Lecture 16 · Projection matrices and least squares

When Ax = b has no solution (too many equations), we settle for the best x̂ — the one that makes Ax̂ as close to b as possible. "Closest" means the projection of b onto the column space of A. ▶ watch on OCW
Strang's example. Three data points (t, b) = (1, 1), (2, 2), (3, 2) that do not lie on a line. We fit the best line b = C + D·t. The system A x = b with A = [[1,1],[1,2],[1,3]] and x = (C, D) is unsolvable, so we solve the normal equations AᵀA x̂ = Aᵀb instead. What you're seeing

Best-fit line (left). The three blue dots are the data. No straight line hits all three, so the fitted line b = C + D·t threads between them. The dashed red vertical segments are the residuals e₁, e₂, e₃ — the gaps between each measured bᵢ and the line's prediction C + D·tᵢ. Least squares chooses C and D to make the sum of the squares of these vertical gaps as small as possible: ‖e‖² = e₁² + e₂² + e₃² is minimized.

Projection picture (right). Stack the three measurements into one vector b = (b₁, b₂, b₃) living in R³. The two columns of A, col₁ = (1,1,1) and col₂ = (t₁,t₂,t₃), span a plane (the column space). b generally pokes out of that plane, which is exactly why Ax = b has no solution. The closest point in the plane is the projection p = Ax̂ (green), and the error e = b − p (red) is perpendicular to the plane. That perpendicularity is the whole game: Aᵀe = 0 ⟺ Aᵀ(b − Ax̂) = 0 ⟺ AᵀA x̂ = Aᵀb.

The worked math

Why projection = least squares

The vector Ax is forced to live in the column space of A — a plane through the origin in R³. Among all points Ax in that plane, the one nearest b is the foot of the perpendicular from b: the projection p. "Nearest" means smallest distance ‖Ax − b‖, and minimizing a distance is the same as minimizing its square ‖Ax − b‖². Since the i-th component of Ax − b is exactly the vertical residual (C + D·tᵢ) − bᵢ, minimizing ‖Ax − b‖² is minimizing the sum of squared vertical errors. The minimizer x̂ is the unique solution of the normal equations AᵀA x̂ = Aᵀb, and p = Ax̂ equals Pb where P = A(AᵀA)⁻¹Aᵀ is the projection matrix onto the column space.

Strang's exact answer

For his data the normal equations are 3C + 6D = 5 and 6C + 14D = 11. Subtracting twice the first from the second gives 2D = 1, so D = 1/2, and back-substitution gives C = 2/3. The best line is b = 2/3 + (1/2)·t. Evaluating it at t = 1, 2, 3 gives the projection p = (7/6, 5/3, 13/6), and the error is e = b − p = (−1/6, 2/6, −1/6). Strang checks that e is perpendicular not just to p but to the entire column space: e·(1,1,1) = −1/6 + 2/6 − 1/6 = 0 and e·(1,2,3) = −1/6 + 4/6 − 3/6 = 0. Why is AᵀA invertible? (the central theory)

Everything above relies on solving AᵀA x̂ = Aᵀb — which only works if AᵀA is invertible. Strang proves the key fact: if the columns of A are independent, then AᵀA is invertible. A square matrix is invertible exactly when its null space is just the zero vector, so it is enough to show AᵀA x = 0 ⟹ x = 0:

  1. Suppose AᵀA x = 0. Dot both sides with x: xᵀAᵀA x = 0.
  2. But xᵀAᵀA x = (Ax)ᵀ(Ax) = ‖Ax‖² — the squared length of Ax. A length is zero only for the zero vector, so Ax = 0.
  3. Now use the hypothesis: if A has independent columns, the only solution of Ax = 0 is x = 0.

So the null space of AᵀA is only {0}, which makes AᵀA invertible. For Strang's data the columns (1,1,1) and (1,2,3) are clearly independent, so AᵀA = [[3,6],[6,14]] is invertible (its determinant 3·14 − 6·6 = 6 ≠ 0) and the best-fit line is unique. The cleanest possible case is when the columns are not just independent but orthonormal (perpendicular unit vectors) — then AᵀA = I, the topic of the next lecture.
