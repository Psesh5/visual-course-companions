Lecture 6 · Column space and nullspace

Two subspaces hide inside every matrix A: the column space C(A) — everything Ax can reach — and the nullspace N(A) — every x with Ax = 0. ▶ watch on OCW
Our matrix. We use a singular 3×3 matrix on purpose, so both subspaces are interesting and both live in R³ where we can draw them: A = [[1,0,1],[2,1,3],[3,1,4]]. Its third column is the sum of the first two (col₃ = col₁ + col₂), so A has rank 2 — not 3.

The intuition

Column space C(A) — what b's are reachable. Multiplying Ax is the same as taking the combination x₁·col₁ + x₂·col₂ + x₃·col₃. So the set of all possible outputs Ax is exactly the set of all combinations of the columns — that is the column space. The three column arrows (left panel) are not independent: col₃ = col₁ + col₂ lies flat in the plane already spanned by col₁ and col₂. So instead of filling all of R³, the columns span only a plane through the origin (shaded blue). The system Ax = b is solvable exactly when b sits on that plane; pick a b poking off the plane and no combination of columns can reach it.

Nullspace N(A) — what gets crushed to zero. Because col₁ + col₂ − col₃ = 0, the vector n = (1, 1, −1) satisfies Ax = 0. Every multiple c·n also gives 0, so the nullspace is the red line through the origin in the direction of n (right panel). The nullspace measures the failure of uniqueness: if x₀ solves Ax = b, then so does x₀ + c·n for every c — a whole line of solutions, parallel to the nullspace.

The dimension count. Here n = 3 columns. There are 2 pivots, so dim C(A) = rank = 2 (a plane). The remaining n − rank = 1 free column gives dim N(A) = 1 (a line). Both subspaces live inside R³: the column space is a 2-D plane of reachable outputs, the nullspace is a 1-D line of inputs sent to zero.

The worked math — elimination

We row-reduce A to find the pivots (they fix C(A)) and the free column (it fixes N(A)).

A = | 1  0  1 |        R2 ← R2 − 2·R1        | 1  0  1 |     R3 ← R3 − R2     | 1  0  1 |
    | 2  1  3 |   -->   R3 ← R3 − 3·R1   -->  | 0  1  1 |       -->           | 0  1  1 | = U
    | 3  1  4 |                               | 0  1  1 |                     | 0  0  0 |

Pivots in columns 1 and 2 (the boxed leading 1's).  Column 3 has NO pivot -> it is FREE.

Column space. The pivot columns are columns 1 and 2, so a basis for C(A) is the original first two columns {(1,2,3), (0,1,1)}. They span the plane. dim C(A) = 2. (Equivalently, the plane has normal col₁ × col₂ = (−1, −1, 1), giving the plane equation −x − y + z = 0; check that all three columns satisfy it.)

Nullspace. From U, with free variable x₃ = 1, back-substitute: row 2 gives x₂ + x₃ = 0 ⇒ x₂ = −1; row 1 gives x₁ + x₃ = 0 ⇒ x₁ = −1. The special solution is (−1, −1, 1) — the same line as n = (1, 1, −1) (just the opposite sign). So N(A) = { c·(1, 1, −1) }, a line; dim N(A) = 1.

The dependency, confirmed. col₁ + col₂ = (1,2,3) + (0,1,1) = (1,3,4) = col₃. Equivalently A·(1,1,−1) = 0 — verified numerically in the status box above once the plots load.

Strang's question: is the union of two subspaces a subspace?

Before the column space, Strang stops to make the central point about what "subspace" means. Take a plane P through the origin and a line L through the origin — each is a subspace of R³. Now form their union P ∪ L (every vector that lies in P or L). He asks the class point-blank: "Is that a subspace?" The answer is no — and the reason is exactly the one rule that defines a subspace: you must be able to add and stay inside. Pick a vector p in the plane and a vector ℓ on the line; their sum p + ℓ normally flies off to a point that is neither in the plane nor on the line. "Because I can't add," as Strang puts it. (The intersection P ∩ L, by contrast, is a subspace — here just the zero vector — since a sum of vectors lying in both stays in both.)

Strang's actual example — the 4×3 matrix in R⁴

In lecture Strang works a 4×3 matrix, not a 3×3: A = [[1,1,2],[2,1,3],[3,1,4],[4,1,5]], i.e. columns col₁ = (1,2,3,4), col₂ = (1,1,1,1), col₃ = (2,3,4,5). Again col₃ = col₁ + col₂. The columns live in R⁴, so we can't draw them — but the logic is identical to the 3×3 picture above, and the numbers are worth seeing because the exam questions use them.

Column space = a 2-D subspace of R⁴. Three columns, but the third is dependent (col₃ = col₁ + col₂), so they span only a 2-dimensional plane sitting inside the 4-dimensional space — not all of R⁴. This is why Ax = b (four equations, three unknowns) usually has no solution: most right-hand sides b poke off that plane. Nullspace = a line in R³. The weights that kill the columns satisfy x₁·col₁ + x₂·col₂ + x₃·col₃ = 0. Since col₁ + col₂ − col₃ = 0, the vector (1, 1, −1) works — and so does every multiple c·(1, 1, −1) (Strang's class offers (1,1,−1), then (2,2,−2), before he says "tell me the whole lot": all of c·(1,1,−1)). The x's have 3 components, so the nullspace is a line through the origin in R³ — exactly the red line drawn in the right panel up top, because this 4×3 matrix has the same dependency as our 3×3 one.

Why the nullspace is genuinely a subspace. Suppose Av = 0 and Aw = 0. Then A(v + w) = Av + Aw = 0 + 0 = 0 (the distributive law), so v + w is in the nullspace too. And A(cv) = c·(Av) = c·0 = 0, so every scalar multiple stays in. Both closure rules hold — and the zero vector is automatically in (take c = 0) — so N(A) earns the word "space." Contrast this with the union P ∪ L above, where adding broke closure.
