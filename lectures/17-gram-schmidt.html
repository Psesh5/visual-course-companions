Intuition — what Gram-Schmidt is really doing

You already trust one vector, a. Make it length one and call it q₁ — that fixes a direction. Now b points off somewhere, but part of it just repeats q₁. That repeated part is the projection (q₁·b)q₁: how much of b already lives along q₁. Subtract it. What survives, B = b − (q₁·b)q₁, has zero overlap with q₁ — it is perpendicular by construction. Normalize it to get q₂.

The third vector c overlaps with both directions we have so far, so we subtract both projections: C = c − (q₁·c)q₁ − (q₂·c)q₂. Whatever is left is perpendicular to the plane of q₁, q₂; normalize to get q₃. At every step the rule is the same — strip out the parts you already have, then rescale to unit length.

Why bother? Orthonormal vectors make life easy. The matrix Q with these columns satisfies Q⊤Q = I, so projections, least squares, and "find the coordinates of any vector" all collapse into simple dot products — no matrix to invert.

Worked math — Gram-Schmidt on a, b, c

A = QR — the factorization hiding in the steps

Every subtraction we did expresses the original vectors back in terms of the q's. Collecting that bookkeeping gives A = QR, where A = [a b c] has the originals as columns, Q = [q₁ q₂ q₃] is orthonormal, and R is upper triangular — triangular precisely because qᵢ is perpendicular to every later original, so qᵢ·(earlier vectors) never contributes. Since Q⊤Q = I, the recipe for R is just R = Q⊤A, and entry Rij = qᵢ·(column j of A).
