The intuition — what makes a map linear

A transformation T is linear when it respects the two operations that build every vector:

Additivity: T(v + w) = T(v) + T(w)
Homogeneity: T(c·v) = c·T(v)

Geometrically those two rules force three visible facts, all on display in the figure:

  • Lines stay lines. A straight segment never curves — combinations map to combinations.
  • The origin is pinned. T(0) = T(0·v) = 0·T(v) = 0. (This is why pure shifts are not linear — they move the origin.)
  • The grid stays evenly spaced and parallel. Equal steps in input become equal steps in output, so parallel lines remain parallel and equally gridded — they may tilt, stretch, or collapse, but never bend.

The deep consequence: a vector is just v = x·e₁ + y·e₂, so by the two rules T(v) = x·T(e₁) + y·T(e₂). Knowing T(e₁) and T(e₂) tells you everything. Stack those two output vectors as columns and you have the matrix: A = [ T(e₁) | T(e₂) ], and then T(v) = A·v.

The worked math

1 · Columns are the images of the basis vectors. Multiply a matrix by e₁ = (1,0):

[ a  b ] [ 1 ]   [ a ]            [ a  b ] [ 0 ]   [ b ]
[ c  d ] [ 0 ] = [ c ] = T(e₁),   [ c  d ] [ 1 ] = [ d ] = T(e₂).

So the first column is T(e₁) and the second column is T(e₂) — no coincidence, it is the definition of matrix–vector multiplication.

2 · Deriving the rotation matrix. Rotate everything counter-clockwise by θ. Track only the two basis vectors:

  • e₁ = (1,0) sweeps onto the unit circle at angle θ: T(e₁) = (cos θ, sin θ).
  • e₂ = (0,1) starts at 90° and rotates to 90°+θ: T(e₂) = (cos(90°+θ), sin(90°+θ)) = (−sin θ, cos θ).

Drop those two images in as columns:

        [  cos θ   −sin θ ]
R(θ) =  [  sin θ    cos θ ]   =  [ T(e₁) | T(e₂) ].

Set the slider to θ = 90°: the columns become (0,1) and (−1,0) — the blue arrow swings up the y-axis and the green arrow swings to the −x direction, matching [[0,−1],[1,0]].

3 · Verifying linearity on an example. Take a shear A = [[1,1],[0,1]] with v = (1,2), w = (3,−1), c = 2.

  • Av = (1·1 + 1·2, 0·1 + 1·2) = (3, 2),   Aw = (3 + (−1), −1) = (2, −1).
  • Additivity: v + w = (4, 1), so A(v+w) = (4+1, 1) = (5, 1), which equals Av + Aw = (3,2) + (2,−1) = (5, 1). ✓
  • Homogeneity: cv = (2, 4), so A(cv) = (2+4, 4) = (6, 4) = 2·(3,2) = c·Av. ✓

A non-example — why "shift by v₀" is NOT linear

Strang's Example two: take the whole plane and shift every vector by a fixed v₀, so T(v) = v + v₀. It looks innocent, but it breaks both rules. The fastest tell: T(0) = 0 + v₀ = v₀ ≠ 0 — a linear map must pin the origin, and this one moves it. Drag v₀ and the scalar c below and watch the gap open up.

Homogeneity wants T(c·v) and c·T(v) to land on the same point. With a shift they differ by (c−1)·v₀ — the red gap. The other classic non-example is length, T(v) = ‖v‖: it sends 0 to 0 and doubling v does double the length, but T(−2v) = 2‖v‖ ≠ −2‖v‖ = c·T(v), so homogeneity fails for negative c.

The derivative IS a linear transformation

Strang's closing example, and a beautiful one: the transformation isn't a map of the plane at all — it acts on polynomials. Let the input space be all p = c₁ + c₂x + c₃x² (basis {1, x, x²}, three-dimensional) and let T = d/dx. The output p′ = c₂ + 2c₃x lives in the two-dimensional space with basis {1, x}. Differentiation obeys both rules — (p+q)′ = p′ + q′ and (c·p)′ = c·p′ — which, as Strang notes, is exactly why you can differentiate any function once you know a short list of derivatives.

p(x) =
c₁ 2   c₂ 1   c₃ -1

The 2×3 matrix. Apply Strang's column rule. The input basis is {1, x, x²} and the output basis is {1, x}:

  • d/dx (1) = 0 = 0·1 + 0·x → column (0, 0).
  • d/dx (x) = 1 = 1·1 + 0·x → column (1, 0).
  • d/dx (x²) = 2x = 0·1 + 2·x → column (0, 2).
        [ 0  1  0 ]   [ c₁ ]   [ c₂  ]
A = [ 0  0  2 ] , and A·[ c₂ ] = [ 2c₃ ]  =  coordinates of p′ in {1, x}.

The 3-input / 2-output sizes force a 2×3 matrix — the same shape Strang predicted for any map from R³ to R². Multiplying the coordinate column (c₁, c₂, c₃) by A drops c₁ (the constant differentiates away), keeps c₂, and doubles c₃ — exactly the power rule, now packaged as a matrix.

The matrix depends on the basis

A subtlety that launches the next lecture: the transformation is a geometric fact about space, but the matrix is its description in coordinates — and coordinates depend on the basis you chose. The same projection or rotation written in a different basis (say the transformation's own eigenvectors) gets a different, usually simpler, matrix. The columns are always "where the basis vectors go," but which basis you write them in is up to you. Choosing the right basis to make A diagonal is exactly the change of basis / diagonalization story coming up.
